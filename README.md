# TeX-Machina

**TeX-Machina**는 VS code용 LaTeX 확장 프로그램입니다. LaTeX 문서를 작성하면서 필요한 기능과, 반복 작업의 간편화를 위해 제작되었습니다. 

---

## 🚀 Features (주요 기능)

### 1. 지능형 계산 엔진 (Symbolic Calculation)
NumPy, SymPy, SciPy 패키지를 기반으로 한 기호 연산 기능입니다. 단순 계산기를 넘어, LaTeX 수식 그 자체를 이해하고 수학적 해답을 제시합니다.

- **사용법**: 수식을 드래그하고 `ctrl+shift+;` (CLI) 호출 후 명령 입력.
- **주요 기능**:
  - `simplify`, `expand`, `factor`: 식 정리의 기본.
  - `diff`, `int`, `limit`: 미적분학의 모든 것. (예: `int > x,0,1`)
  - `solve`, `ode`, `pde`: 방정식 및 미분방정식 풀이.
  - `laplace`, `fourier`: 공학 수학의 필수 변환 지원.
  - `dimcheck`: 물리 법칙의 차원(L, M, T)이 맞는지 검사.

### 2. 행렬 자동 생성기 (Matrix Builder)
복잡한 `&`와 `\\`의 지옥에서 해방되세요.
- **명령**: `matrix > 1,2 / 3,4` -> $\begin{bmatrix} 1 & 2 \\ 3 & 4 \end{bmatrix}$ 즉시 생성.
- **분석**: `/ analyze` 옵션을 붙이면 행렬식과 역행렬을 웹뷰에서 바로 확인 가능.

### 3. 고품질 시각화 (Plotting & Visualization)
- **2D**: `PGFPlots` 코드를 생성하여 LaTeX 내에 고해상도 벡터 그래픽 삽입.
- **3D**: `X3DOM` 엔진을 통한 대화형 3D 그래프. PDF/PNG로 내보내기 지원.
- **Complex**: 복소함수의 도메인 컬러링 시각화.

### 4. 스마트 워크플로우 도구
- **OEIS 검색**: 수열의 다음 항이 궁금할 때 바로 검색.
- **스마트 인용 (Cite)**: DOI만 입력하면 BibTeX를 자동 생성하고 `.bib` 파일에 추가.
- **수식 너비 분석**: "Overfull \hbox" 에러를 컴파일 전에 미리 감지.

## 🛠 Macro (매크로 시스템)

TeX-Machina의 매크로는 단순한 텍스트 치환이 아닙니다. **컨텍스트(Context)**를 읽습니다.

- **컨텍스트 인지**: 수식 안에서 `;r`을 치면 계산(`calc`)이 돌아가고, 텍스트에서 `;r`을 치면 논문 인용(`cite`)이 돌아가도록 설정할 수 있습니다.
- **정의**: `define:명령어:이름:컨텍스트`
- **호출**: `;이름`

## 💎 Query (초정밀 쿼리 가이드: AST 조작의 정수) ★★★★★

> **"LaTeX 코드를 텍스트가 아닌 객체(Object)로 다루십시오."**

쿼리 기능은 TeX-Machina의 자부심입니다. 수천 줄의 LaTeX 문서 속에서 특정 구조를 찾아내고, 값을 계산하고, 위치를 재배열하는 모든 과정이 마치 데이터베이스를 쿼리하듯 단 한 줄로 끝납니다. 이 가이드는 여러분을 쿼리 마스터로 만들어 드릴 것입니다.

---

### 1. 쿼리 엔진의 철학 (The Vision)

LaTeX 문서는 수많은 중괄호 `{ }`와 환경 `\begin{...}`으로 얽혀 있습니다. 기존의 '찾기 및 바꾸기'는 중첩된 구조를 파괴하거나 엉뚱한 곳을 수정하기 일쑤였습니다. 

TeX-Machina의 **쿼리 엔진**은 문서를 **추상 구문 트리(AST)**로 파싱합니다. 즉, 엔진은 "이 `{ }`는 이 명령어의 2번째 인자이고, 이 이미지는 이 피겨 환경의 자식이다"라는 사실을 완벽히 이해합니다. 이를 통해 우리는 텍스트 에디터가 아닌 **'문서 수술 도구'**를 손에 넣게 됩니다.

---

### 2. 문법의 기초 체력 (The Anatomy)

쿼리문은 항상 문두에 `;`(최신 권장) 또는 `?`(구버전 호환)를 붙여 시작합니다.

```
; [명령어] -[옵션] [대상][필터] [연산자] [작업][필터] | [파이프라인]
```

#### 2.1 따옴표의 치명적인 차이 (Quotes Magic) ★
이 규칙을 이해하지 못하면 쿼리는 예기치 못한 동작을 할 수 있습니다. 파서는 따옴표를 통해 '의도'를 파악합니다.

- **작은따옴표 (`' '`)**: **"이것을 구조적으로 탐색하라"**는 뜻입니다. 공백이나 특수 문자가 섞인 경로를 묶을 때 필수입니다.
  - 예: `'figure > center'` (가운데 정렬된 피겨를 찾아라)
- **큰따옴표 (`" "`)**: **"이 글자 자체를 문서에 써라"**는 뜻입니다. 삽입될 텍스트를 의미합니다.
  - 예: `>> "Figure 1: "` (타겟을 저 글자로 바꿔라)
- **따옴표 없음**: 엔진은 이를 **"수학적 연산"**이나 **"변수/속성 접근"**으로 해석합니다.
  - 예: `>> #w * 0.5` (너비 속성값에 0.5를 곱해라)

---

### 3. 정밀 타격 선택자 (Selectors)

원하는 요소를 핀포인트로 집어내는 법을 익혀봅시다.

#### 3.1 태그 (`@`) 시스템 (The Shortcuts)
복잡한 이름을 다 치지 않아도 됩니다.
- **`@img`**: `\includegraphics` 명령어.
- **`@fig`**: `\begin{figure} ... \end{figure}` 환경 전체.
- **`@tbl`**: `tabular`, `longtable` 등 모든 표 환경.
- **`@eq`**: `equation`, `align`, `gather` 등 수식 환경 통칭.
- **`@cell`**: 표 내부의 개별 칸 하나하나.
- **`@math`**: 모든 수식 모드 (인라인 `$ $` 포함).

#### 3.2 속성 태그 (`#`) (Property Probing)
명령어의 대괄호 `[ ]` 옵션 내부를 현미경처럼 들여다봅니다.
- **`#w`**: width (너비)
- **`#h`**: height (높이)
- **`#s`**: scale (축척)
- **`#c`**: color (색상)
- **`#scale`**: width, height, scale을 통칭하는 지능형 속성.

#### 3.3 유사 클래스 (Pseudo-classes) (Fine-tuning)
- **`:first` / `:last`**: 결과 중 첫 번째 또는 마지막 요소만 선택.
- **`:nth(n)`**: n번째 요소를 선택 (예: `@cell:nth(3)` - 3번째 칸).
- **`:*` (Content Selector) ★**: 껍데기를 제외한 **내용물**만 타겟팅.
  - `; figure:*` 라고 하면 `\begin{figure}` 태그는 건드리지 않고 내부 코드만 가져옵니다.
  - `; \caption{$$} >> "Fig: $$"` 처럼 중괄호 안쪽 내용만 유지할 때 필수입니다.

---

### 4. 연산자 대백과사전 (Operators Encyclopedia)

#### 4.1 계층 및 탐색 (The Pathfinders)
문서의 가계도를 따라 이동합니다.
- **`>` (직계 자식)**: `figure > @img` (피겨 바로 밑에 이미지가 있는 경우만)
- **`~` (형제)**: `\includegraphics ~ \caption` (나란히 있는 이미지와 캡션)
- **`...` (심층 탐색)**: `figure ... \caption` (피겨 안 깊숙한 곳 어딘가에 있는 캡션)
- **`$` (강제 컨텍스트)**: `figure$@img` (피겨 환경이라는 바운더리를 먼저 확정한 뒤 내부를 뒤집니다. 연산 속도가 가장 빠릅니다.)
- **`_` (자기 참조)**: 현재 잡힌 요소 자신. `_{#w}`는 자신의 너비를 의미합니다.

#### 4.2 변이 및 할당 (The Shapers)
현실(문서)을 바꿉니다.
- **`>>` (치환/덮어쓰기)**: 타겟을 우항으로 바꿉니다. (우항에 따옴표가 없으면 수식 연산!)
- **`:=` (대입)**: 속성값을 명시적으로 고정합니다. (예: `; @img >> #w := 10cm`)
- **`+=` / `-=` (수치 증감) ★★★**: **상대 수치 연산 지원.**
  - `; @img >> #w += 10%` : "너비가 얼마든 10%만 키워줘."
  - 엔진은 내부적으로 `0.5\textwidth`를 분석하여 `0.55\textwidth`로 정확하게 계산해냅니다.
- **`><` (감싸기)**: `; @img >< "center"` (이미지를 center 환경으로 포장)
- **`<>` (벗기기)**: 감싸고 있는 환경 껍데기만 삭제.
- **`^^` / `vv`**: 요소를 부모 밖으로 빼거나 자식 안으로 넣음.

#### 4.3 특성(Trait) 연산자 (Instant Styling)
파이프라인 뒤에 붙여 즉시 서식을 입힙니다.
- **`+ bold`**: `\textbf{$$}` 적용.
- **`+ italic`**: `\textit{$$}` 적용.
- **`- clear`**: 모든 bold, italic 서식을 지우고 순수 텍스트로 복구.

---

### 5. 메모리와 변수: "지능형 동기화"

복잡한 문서 편집의 핵심은 '기억'입니다.

- **`-> //1`**: 결과를 1번 레지스터에 저장.
- **`as $var`**: 결과를 `$var` 라는 이름의 변수에 저장.

#### ★ 지역별 스코프 동기화 (Lexical Scope Synchronization)
이 기능은 TeX-Machina 쿼리의 가장 위대한 발명입니다. 
문서에 이미지가 10개, 캡션이 10개 있다고 칩시다. `; move $cap >> $img.|` 라고 명령하면, 엔진은 바보처럼 1번 캡션을 1번 이미지에 붙이는 게 아닙니다.
**"이 캡션과 이 이미지가 같은 figure 환경(부모) 안에 들어있는가?"**를 먼저 검증합니다. 같은 가족(Scope)인 것들끼리만 짝을 지어 이동시킵니다. 이로써 챕터 1의 캡션이 챕터 2의 이미지로 날아가는 사고를 완벽히 방지합니다.

---

### 6. 흐름 제어 (The Conductor)

- **`&` (비동기)**: 두 명령을 독립적으로 동시에 실행.
- **`&&` (동기)**: 앞 명령이 문서를 완전히 고친 후, 그 결과물 위에서 다음 명령 실행. (안전제일!)
- **`|` (파이프라인)**: 앞선 선택 결과를 그대로 들고 다음 작업으로 이동.
- **`loop { ... }`**: 결과가 더 이상 없을 때까지 무한 반복.
  - `#i`: 전체 루프 카운터 (1, 2, 3...)
  - `#j`: 현재 배열 내 순번.

### 7. 끝판왕 실전 예시 (Masterclass Scenarios)

#### 🎓 시나리오 1: 모든 이미지 경로 및 크기 일괄 교정
**도전**: 논문의 모든 이미지 파일명을 `img/` 폴더 안으로 옮겼습니다. 또한 저널 규격에 맞춰 모든 이미지 너비를 기존 대비 15% 줄이고 싶습니다.
```
; @img { >> "img/$$" , >> #w -= 15% }
```
- **해설**: `{ }`로 한 타겟에 두 작업을 시킵니다. `$$`는 원래 파일명을 기억하고, `-= 15%`는 너비를 상대적으로 계산해 깎아냅니다.

#### 🎓 시나리오 2: 표(Table) 서식 싹 지우고 헤더만 강조하기
**도전**: 인터넷에서 긁어온 표들의 서식이 제각각입니다. 이탤릭, 굵게 등이 섞여 있습니다. 싹 다 지우고 첫 줄(헤더)만 굵게 만들겠습니다.
```
; @tbl$ @cell | - clear && @tbl$ @row[1] > @cell | + bold
```
- **해설**: `&&`를 써서 '전체 서식 제거'를 완료한 후, 다시 표를 찾아 첫 번째 행(`@row[1]`)만 골라 `+ bold`를 먹입니다.

#### 🎓 시나리오 3: 캡션 누락 그림만 찾아 자동 조치하기
**도전**: 수백 개의 그림 중 캡션을 깜빡한 그림들을 찾아, 중앙 정렬을 해주고 "[TODO: 캡션 작성]" 이라는 문구를 박아 넣습니다.
```
; @fig without \caption | move ^(@img)^ >> figure.| & <+ "\centering\n" & +> "\caption{[TODO: 캡션 작성]}"
```
- **해설**: `without`으로 타겟을 좁히고, 파이프라인(`|`)으로 넘깁니다. 서브쿼리(`^()^`)로 이미지를 피겨 내부 하단(`figure.|`)으로 옮기며 앞뒤에 코드를 삽입(`&`)합니다.

#### 🎓 시나리오 4: 수식 번호(Equation) 순차 자동 라벨링
**도전**: 수식들에 `\label{eq:1}`, `\label{eq:2}` 식으로 번호를 매겨야 합니다.
```
; find @eq > \label{$$} >> "eq:#j"
```
- **해설**: `#j` 변수는 배열의 순서를 뜻합니다. 별도의 반복문 없이도 모든 라벨의 내용물을 1, 2, 3 순번으로 덮어씁니다.

#### 🎓 시나리오 5: 분모가 0인 위험한 수식 검출 및 수정
**도전**: `\frac{...}{0}`은 컴파일 에러나 수학적 오류를 냅니다. 분모가 0인 놈들을 찾아 1로 바꿉니다.
```
; find \frac{@arg[2]:@int} where @int == 0 >> 1
```
- **해설**: `@arg[2]`는 분모입니다. 이 중 정수(`@int`)가 정확히 `0`인 노드만 골라 `1`로 치환합니다.

### 8. 꿀팁 및 주의사항 (Troubleshooting)

1.  **"쿼리가 왜 안 되죠?"**: 따옴표를 확인하세요. 글자 그대로를 넣고 싶다면 반드시 큰따옴표(`" "`)로 감싸야 합니다.
2.  **"문서가 엉뚱하게 바뀌었어요"**: `&` 대신 `&&`를 써보세요. 문서 구조가 바뀔 때는 앞 작업이 완전히 끝난 후 다음 작업을 하는 것이 훨씬 안전합니다.
3.  **커서의 미학**:
    - `.|figure` : 피겨 시작줄 **앞**
    - `.|figure|` : 피겨 시작줄 **안쪽(첫 줄)**
    - `figure.|` : 피겨 끝줄 **안쪽(마지막 줄)**
    - `|figure.` : 피겨 끝줄 **뒤**
4.  **성능 최적화**: 너무 큰 파일에서는 `find @img` 보다는 `figure$@img` 처럼 컨텍스트를 좁혀주는 것이 좋습니다.

## 🏗 Installation & Setup (설치 및 설정)

### 필수 요구 사항
- **Python 3.x**: 수치 연산과 지능형 엔진 구동을 위해 필수적입니다.
- **필수 패키지 설치**:
  ```bash
  pip install sympy numpy scipy matplotlib
  ```

### 설치 방법
1. VS Code 확장 마켓플레이스에서 `TeX-Machina`를 검색하여 설치합니다.
2. `.tex` 파일을 열고 `ctrl+shift+;` (macOS: `cmd+shift+;`)를 눌러 명령줄(CLI)이 뜨는지 확인하세요.
