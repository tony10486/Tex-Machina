## 개선이 핑요한 사항
- quickpick을 통한 명령어 입력 시 뜨는 연관 명령어를 모든 기능에 대해 지원해야 함 
- 설정에서 더 많은 커스터마이징이 가능하도록 해야 함 
- 수식 블록 기반 히스토리 : 수식 부분에서 히스토리 기능을 지원해서, 수식에 한정해 git과 같은 수정 내역 관리 시스템을 만들어야 함

- 수식 분할 기능에서 덧셈 기호에 대해서 줄바꿈을 할 수 있도록 옵션을 마련하면 좋을 듯 함
- table generator 기능을 더 수정했으면 함

1. 고급 필터 및 논리 연산 (가장 큰 격차)
  현재는 태그 이름과 계층 구조(>)만으로 대상을 찾습니다. README에 있는 다음 기능들은 아직 작동하지 않습니다.
   * 속성/상태 필터 (where, without, has):
       - ; @fig without \caption (캡션 없는 그림만 찾기)
       - ; \section[title="Intro"] (제목이 "Intro"인 섹션만 찾기)
   * 논리 결합 (and, or, not):
       - ; @img and #w > 10cm (너비가 10cm 이상인 이미지)


  2. 복잡한 문서 탐색 (Advanced Navigation)
  현재는 >(직계 자식)만 지원하며, 다음의 경로 탐색은 불가능합니다.
   * ~ (형제 탐색): ; \includegraphics ~ \caption (이미지 다음에 오는 캡션 찾기)
   * ... (심층 탐색): ; figure ... \caption (피겨 내부 어디든 있는 캡션 찾기)
   * <, << (부모/조상 탐색): ; \includegraphics < figure (이미지를 감싸고 있는 피겨 찾기)


  3. 구조적 명령어 (Structural Commands)
  현재는 내용을 바꾸거나(>>) 앞뒤에 붙이는(+>, <+) 작업 위주이며, 다음의 구조적 변경은 지원하지 않습니다.
   * move: 대상을 다른 위치(예: 부모의 끝)로 옮기기.
   * exchange / <=>: 두 요소의 위치나 내용 서로 바꾸기.
   * duplicate: 요소 복제하기.
   * extract: 특정 부분만 남기고 나머지는 삭제하기.


  4. 표(Table) 내부 정밀 제어
  현재 SemanticScanner는 표의 셀(Cell)이나 행(Row)을 개별 노드로 인식하지 못합니다.
   * @cell, @row: 표 내부의 특정 칸이나 줄을 타겟팅하는 기능.
   * order by: 내용이나 너비 기준으로 요소(주로 표의 행) 정렬하기.


  5. 상태 유지 및 변수 (Memory & Variables)
   * 레지스터 (-> //1): 결과를 메모리에 저장했다가 나중에 꺼내 쓰는 기능.
   * 사용자 변수 (as $var): 특정 결과를 변수로 이름 붙여 관리하기.
   * 루프 카운터 (#i, #j): 여러 요소를 처리할 때 번호를 매기는 기능. (예: \label{eq:#j})


  6. 중첩 및 제어 흐름
   * loop { ... }: 특정 조건이 만족될 때까지 반복 실행.
   * ^( ... )^ (Subquery): 쿼리 문법 안에 또 다른 쿼리를 넣어 필터 조건으로 사용.
   * 스코프 동기화: 같은 환경 내의 요소끼리 자동으로 짝을 지어 처리하는 지능형 매칭.
