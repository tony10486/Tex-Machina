1. 고급 셀렉터 및 탐색 연산자 (Hierarchy & Search)
  현재는 평면적인 이름 검색만 가능합니다. 문서의 구조를 따라가는 연산자들이 필요합니다.
   * 계층 연산자: > (직계 자식), ~ (형제), ... (심층 탐색), < / << (부모/조상)
   * 컨텍스트 제한: $ (특정 환경 내부로 검색 범위 한정)
   * 자기 참조: _ (현재 선택된 요소 자체의 속성 접근)
   * 서브쿼리: ^( ... )^ (필터 조건 안에 또 다른 쿼리 삽입)


  2. 논리 필터 및 속성 분석 (Logic & Filters)
  단순 선택을 넘어, 요소의 '상태'를 판별하는 로직이 빠져 있습니다.
   * 상태 필터: where, without, has (자식 요소 유무나 속성값 조건 검사)
   * 논리 결합: and, or, not (여러 조건을 조합)
   * 정렬: order by (검색 결과를 너비나 내용 순으로 정렬)
   * 태그 매핑: @img, @fig, @tbl, @eq, @cell, @math 등 숏컷 태그의 체계적 정의


  3. 구조적 변형 연산자 (Structural Manipulation)
  단순히 글자를 바꾸는 >> 외에, LaTeX 구조를 조작하는 연산자가 필요합니다.
   * 위치 조작: move, exchange, duplicate, extract 명령어 구현
   * 환경 조작: >< (환경으로 감싸기), <> (환경 제거), >+< (텍스트로 감싸기)
   * 계층 이동: ^^ (한 단계 밖으로 추출), vv (자식 요소로 삽입)
   * 위치 교체: <-> (두 요소의 위치 스왑), <=> (내용물만 스왑)


  4. 상태 관리 및 변수 시스템 (State & Memory)
  복잡한 자동화를 위한 데이터 저장 기능이 필요합니다.
   * 레지스터: -> //1 (결과를 메모리에 저장)
   * 사용자 변수: as $var (이름을 붙여 저장)
   * 자동 카운터: #i (전체 순번), #j (현재 그룹 내 순번)
   * 스코프 동기화: 서로 다른 요소를 매칭할 때 같은 부모 아래 있는지 확인하는 동기화 로직


  5. 파서 및 흐름 제어 고도화 (Parser & Control)
  쿼리문의 '의도'를 정확히 해석하고 실행 순서를 제어해야 합니다.
   * 따옴표 엔진: ' ' (구조적 경로)와 " " (리터럴 텍스트)를 엄격히 구분하는 파싱 로직
   * 수치 연산: 따옴표 없는 우항을 수학식으로 인식하여 계산 (#w * 0.5 등)
   * 반복문: loop { ... } (조건이 만족될 때까지 반복 실행)
   * 실행 제어: & (비동기/병렬)와 && (순차 실행) 구분
   * 정밀 커서: .|target, .|target|, target.|, |target. (총 4가지 정밀 삽입 위치)
